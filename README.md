# 📦 FinTech Delivery Optimization & Support Load Analysis

## 📌 О проекте
Анализ процесса доставки финансовых продуктов клиентам. Проект направлен на выявление узких мест в логистике (Delivery) и оценку их влияния на нагрузку службы поддержки (Customer Care).

## 🎯 Бизнес-задачи
1. Рассчитать метрику **Delay Rate** (доля доставок с опозданием >30 минут) в разрезе городов.
2. Оценить влияние качества логистики на генерацию тикетов в саппорт (категория "Где курьер?").
3. Найти инсайты для оптимизации костов.

## 🛠 Стек технологий
* **SQL (PostgreSQL):** CTE, Aggregations, Date/Time functions, Conditional Logic (CASE).

## 💡 Главные инсайты (Insights to Action)
* **Анализ данных показал:** Клиенты, к которым курьер опаздывает более чем на 30 минут, генерируют в X раз больше обращений в Customer Care в тот же день. 
* **Продуктовое действие (Action):** Внедрить автоматический пуш-скрипт: если расчетное время прибытия курьера сдвигается на 15+ минут, клиенту автоматически отправляется уведомление с извинениями и новым тайм-слотом. Это позволит превентивно снять тревожность клиента и разгрузить первую линию поддержки.

## 📂 Структура репозитория
* `1_init_schema_and_data.sql` — DDL и DML скрипты для развертывания тестовой базы.
* `2_delivery_analytics.sql` — SQL-запросы с аналитикой и расчетом продуктовых метрик.
